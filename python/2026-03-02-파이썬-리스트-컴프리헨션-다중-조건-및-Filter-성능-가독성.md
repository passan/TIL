### Context
* 파이썬에서 시퀀스 데이터(예: 리스트)를 특정 조건에 따라 필터링할 때, 리스트 컴프리헨션(List Comprehension)은 간결하고 강력한 문법을 제공합니다.
* 하지만 여러 개의 `if` 조건을 한 줄에 나열할 경우 가독성이 저하될 수 있으며, 이로 인해 `filter()` 함수를 사용하는 방안이 대안으로 고려됩니다.
* 본 문서는 리스트 컴프리헨션의 다중 조건 사용과 `filter()` 함수의 가독성 및 성능 측면을 비교하고, 적절한 활용 가이드라인을 제시합니다.
* 파이썬 버전: 3.8 이상 (최신 안정 버전).

### Core
* **리스트 컴프리헨션 (List Comprehension) 예시 - 다중 `if` 조건**
```python
# 10보다 크고 짝수인 숫자만 필터링
numbers = list(range(1, 21))
filtered_numbers_comprehension = [x for x in numbers if x > 10 if x % 2 == 0]
print(f"List Comprehension 결과: {filtered_numbers_comprehension}")

# 또는 다음과 같이 `and` 연산자로 결합할 수 있습니다.
filtered_numbers_comprehension_and = [x for x in numbers if x > 10 and x % 2 == 0]
print(f"List Comprehension (and) 결과: {filtered_numbers_comprehension_and}")
```

* **`filter()` 함수 예시 - 다중 조건**
```python
# 10보다 크고 짝수인 숫자만 필터링
numbers = list(range(1, 21))
def is_greater_than_10_and_even(num):
    return num > 10 and num % 2 == 0

# 또는 람다 함수 사용
filtered_numbers_filter = list(filter(lambda x: x > 10 and x % 2 == 0, numbers))
print(f"filter() 함수 결과: {filtered_numbers_filter}")
```

### Insight
* **가독성 (Readability)**
  * 리스트 컴프리헨션은 단일 또는 2개 정도의 간단한 조건에서는 매우 직관적이고 파이써닉(Pythonic)합니다.
  * 하지만 여러 개의 `if` 절이 연달아 오거나 조건이 복잡해지면, 가독성이 떨어질 수 있습니다. 이 경우 `and` 연산자를 사용하여 조건을 하나로 묶는 것이 좋습니다.
  * `filter()` 함수는 별도의 함수(named function)나 람다(lambda) 함수를 사용하여 필터링 로직을 분리할 수 있어, 복잡한 조건을 처리할 때 가독성 측면에서 유리할 수 있습니다.

* **성능 (Performance)**
  * 일반적으로 리스트 컴프리헨션은 C로 구현되어 있고 내부적으로 최적화되어 있어 `filter()`와 람다 함수 조합보다 빠른 경향이 있습니다.
  * `filter()`는 함수 호출 오버헤드(function call overhead)가 발생하므로, 대량의 데이터를 처리할 때는 리스트 컴프리헨션이 성능상 이점을 가질 수 있습니다.
  * 이는 특히 간단한 조건일수록 더욱 두드러지며, 조건 로직이 복잡해질수록 성능 차이는 줄어들 수 있습니다.

* **가이드라인 (Guideline)**
  * **비전공자 또는 코드 가독성이 최우선인 경우**: `filter()` 함수와 명명된 함수(named function) 또는 가독성 높은 람다 함수를 사용하여 조건을 명확하게 분리하는 것을 권장합니다.
  * **성능이 중요하거나 데이터 처리량이 많은 경우**: 리스트 컴프리헨션을 사용하고, 필요하다면 `and` 연산자로 조건을 결합하여 한 줄의 `if` 절로 만드는 것이 효율적입니다.
