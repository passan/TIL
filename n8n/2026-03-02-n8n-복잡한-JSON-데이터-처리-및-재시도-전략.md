### Context
* 외부 API 연동을 통해 수집된 JSON 데이터는 종종 깊게 중첩된 구조를 가지며, 원하는 특정 필드의 값을 추출하는 데 어려움이 발생한다. 특히 n8n 워크플로우 내에서 이러한 복잡한 데이터를 효율적으로 처리하기 위해서는 JavaScript 기반의 데이터 가공 로직이 필수적이다.
* 또한, 외부 API 호출 시 네트워크 문제, 서비스 불안정 등 다양한 원인으로 인해 에러가 발생할 수 있다. 이러한 에러 상황을 안정적으로 처리하고 실패한 작업을 자동으로 재시도(Retry)하는 예외 처리 메커니즘을 구축하는 것은 견고한 자동화 워크플로우 설계에 있어 매우 중요하다. 본 문서는 n8n 워크플로우에서 복잡한 JSON 데이터 처리 및 에러 재시도 전략을 다룬다.

### Core
```javascript
// n8n Code 노드에서 JSON 데이터 추출 예시
// 'item.json'은 현재 아이템의 전체 JSON 데이터를 나타냅니다.

// 예시 1: 단일 깊이의 필드 추출
// const extractedValue = item.json.dataField;

// 예시 2: 중첩된 객체 내의 필드 추출
// const nestedValue = item.json.level1.level2.targetField;

// 예시 3: 배열 내의 첫 번째 객체에서 필드 추출
// if (item.json.dataArray && item.json.dataArray.length > 0) {
//   const arrayFirstItemValue = item.json.dataArray[0].specificField;
//   return { json: { extracted: arrayFirstItemValue } };
// }

// 예시 4: 복잡한 JSON 데이터를 순회하며 특정 조건에 맞는 데이터 추출
// 가상의 JSON 구조:
/*
{
  "status": "success",
  "results": [
    {
      "id": "item1",
      "details": {
        "name": "Product A",
        "price": 100
      }
    },
    {
      "id": "item2",
      "details": {
        "name": "Product B",
        "price": 200
      }
    }
  ]
}
*/
const inputData = item.json;
let products = [];

if (inputData && inputData.status === "success" && inputData.results) {
  for (const result of inputData.results) {
    if (result.details && result.details.name && result.details.price) {
      products.push({
        productId: result.id,
        productName: result.details.name,
        productPrice: result.details.price
      });
    }
  }
}

// 추출된 데이터를 n8n의 다음 노드로 전달
return { json: { products: products } };
```
*   **에러 처리 및 재시도 전략 (Error Handling and Retry Strategy)**
    *   n8n에서 API 요청 노드(예: HTTP Request 노드)는 자체적인 재시도 설정을 포함하는 경우가 많다. 노드 설정에서 'Retry on Fail' 옵션을 활성화하고 재시도 횟수(Retries)와 재시도 간격(Retry Interval)을 설정하여 간단한 네트워크 오류 등에 대응할 수 있다.
    *   더 복잡한 에러 처리 로직이나 특정 에러 코드에 대한 조건부 재시도가 필요한 경우, 'Error Trigger' 노드와 'If' 노드를 활용한 별도의 에러 워크플로우를 구성한다.
        *   **메인 워크플로우**: 에러 발생 가능성이 있는 노드에 'Continue On Error' 옵션을 활성화하여 워크플로우가 중단되지 않도록 한다.
        *   **Error Trigger 워크플로우**: 'Error Trigger' 노드를 시작으로 하는 별도의 워크플로우를 생성한다. 이 워크플로우는 메인 워크플로우에서 발생한 에러 이벤트를 수신한다.
        *   **재시도 로직**: 'Error Trigger'로 전달된 에러 정보(예: `error.json.executionId`)를 사용하여 'Execute Workflow' 노드를 통해 실패한 메인 워크플로우의 실행을 재시도할 수 있다. 필요에 따라 'Wait' 노드를 사용하여 일정 시간 대기 후 재시도할 수 있다.

### Insight
*   n8n의 'Code' 노드를 활용한 JavaScript는 복잡한 중첩 JSON 데이터 구조에서 필요한 정보를 유연하게 추출하고 변환하는 강력한 도구다. `item.json` 객체에 접근하여 점 표기법(`.`)이나 대괄호 표기법(`[]`)을 통해 원하는 필드를 정확히 지정할 수 있다. 특히 배열 내 객체 처리는 `map`, `filter`, `forEach`와 같은 배열 메서드를 활용하면 더욱 효율적이다.
*   안정적인 워크플로우 운영을 위해서는 에러 처리가 필수적이다. HTTP Request 노드의 내장 재시도 기능은 기본적인 실패에 효과적으로 대응하며, 'Error Trigger' 노드를 활용한 별도의 에러 워크플로우는 특정 조건(예: HTTP 5xx 에러, 특정 응답 메시지)에 따른 맞춤형 재시도 로직이나 알림 기능 구현을 가능하게 하여 워크플로우의 견고성(Robustness)을 크게 향상시킨다. 이는 잠재적인 데이터 손실을 방지하고 자동화 시스템의 신뢰도를 높이는 핵심 전략이다.
